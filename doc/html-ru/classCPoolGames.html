Подробное описание

Контракт периодического розыгрышей вознаграждения на пулах депозитов в фермы

Необходимо сделать:

Подтверждение победителя другими участниками пула (кроме победителя) за небольшое вознаграждение из выигрыша первым N подтвердившим

Выбор участниками токена или XTZ, в котором они хотят получить выигрыш и автоконвертация выигрыша в нужный конкретному пользователю токен

По флагу, установленному пользователем, автоинвестирование выигранных токенов в тот же пул через автоконвертацию

Получение и вывод менеджерами пула вознаграждения с момента приостановки партий и до извлечения пользователями всех депозитов

Старт розыгрыша при внесении первого депозита в пул

Кроме собственно алгоритма в каждом случае используются дополнительные настройки minDeposit, minSeconds, maxDeposit (только для AlgoTimeVol) для дополнительных ограничений

См. также
t_opts.minDeposit, t_opts.minSeconds, t_opts.maxDeposit
Элементы перечислений
AlgoTime 

Вероятность выигрыша пропорциональна суммарному времени в игре

Время вступления каждого пользователя в розыгрыш фиксируется. В конце партии суммируются все времена пребывания в партии всех пользователей (и эта сумма принимается за вероятность 1.0). Далее случайное число нормируется к этой сумме и выбирается пользователь на основании этого числа. В итоге вероятность выигрыша пользователя пропорциональна его времени нахождения в пуле. Обнуление депозита удаляет пользователя из партии, соответственно удаляет его время нахождения в пуле, это делает бессмысленным выход и повторный вход в партию. Этот алгоритм более интересен пользователям с малыми депозитами, вероятность выигрыша не зависит от размера депозита, поэтому можно вложить мало, а выиграть много.

См. также
t_opts.minDeposit, t_opts.minSeconds, t_weight, t_game.weight
AlgoTimeVol 

Вероятность выигрыша пропорциональна сумме произведений объема на время в игре

Время вступления каждого пользователя в розыгрыш и его депозит на этот момент фиксируется. При пополнении депозита, предыдущий депозит умножается на его время присутствия в пуле в секундах и эта сумма копится отдельно по каждому пользователю. Это позволяет пользователям увеличивать вероятность выигрыша, увеличивая депозит во время партии. В конце партии суммируются все времена пребывания в партии умноженные на депозиты всех пользователей (с учетом пополнений депозита в течение партии, см. выше) (и эта сумма принимается за вероятность 1.0). Далее случайное число нормируется к этой сумме и выбирается пользователь на основании этого числа. В итоге вероятность выигрыша пользователя пропорциональна сумме произведений его депозита и времени нахождения в пуле его ликвидности. Обнуление депозита удаляет пользователя из партии, соответственно удаляет его время нахождения в пуле, это делает бессмысленным выход и повторный вход в партию. Этот алгоритм более интересен пользователям с большими депозитами, вероятность выигрыша зависит от размера депозита, поэтому можно существенно увеличить вероятность выигрыша, вложив большой депозит.

См. также
t_opts.minDeposit, t_opts.maxDeposit, t_weight, t_game.weight
AlgoEqual 

Вероятность выигрыша равновероятна

Вероятность выигрыша равновероятна для всех пользователей в пуле, которые присутствуют на окончание партии, у всех вес 1. Использование алгоритма без minSeconds уязвимо перед халявщиками, которые входят в пул только перед розыгрышем

См. также
t_opts.minSeconds, t_weight, t_game.weight

◆ t_game_state

enum CPoolGames::t_game_state

Состояния партии розыгрыша вознаграждения

Элементы перечислений
GameStateActivating 

Идет запуск партии

Это состояние остается в хранилище только когда пользователи вносят или извлекают депозит после окончания розыгрыша в пустом пуле. Запуск нового розыгрыша требует дополнительных операций некорректно будет расход газа для этих операций истребовать с пользователя. Полноценный запуск партии будет произведен позднее менеджером пула

GameStateActive 

Идет партия розыгрыша

GameStateComplete 

Партия розыгрыша закончена по времени

GameStateWaitRandom 

Партия розыгрыша закончена по времени, ожидаем случайное число для определения победителя

GameStateWaitWinner 

Партия розыгрыша закончена по времени, случайное число получено (или не требуется), ожидаем определения победителя внешним кодом

GameStatePause 

Партии приостановлены (партия розыгрыша завершена, но запуск следующей заблокирован)

PoolStatePause 

Пул приостановлен

Если партия розыгрыша активна, она продолжается до завершения, но следующая не будет запущена.

См. также
PausePool, CreatePool
PoolStateRemove 

Пул на удаление

Если партия розыгрыша активна, она продолжается до завершения. По окончании партии, когда пользователи заберут все депозиты, пул будет удален во время списания последнего депозита. Если же пул уже пуст на момент установки состояния, он будет удален немедленно.

См. также

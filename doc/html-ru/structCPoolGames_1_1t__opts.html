◆ feePercent

nat CPoolGames::t_opts::feePercent

Процент комиссии за розыгрыш

В интервале [0; 100], в сумме с другими процентами должно быть 100. Для осуществления розыгрыша админ контракта должен вызвать контракт после завершения партии, запросить случайное число у оракула и осуществить розыгрыш, что потребует затрат на выполнение.

См. также
winPercent, burnPercent

◆ maxDeposit

nat CPoolGames::t_opts::maxDeposit

Максимальный депозит для пула

Пул с алгоритмом AlgoTimeVol может позволить владельцу огромного депозита войти в последний момент и с большой вероятностью забрать вознаграждения. Чтобы ограничить размеры депозитов в пуле разумными рамками и дать пользователям сопоставимые шансы этот параметр вместе с minDeposit позволит получить честный розыгрыш. 0 - нет ограничения

См. также
AlgoTimeVol

◆ minDeposit

nat CPoolGames::t_opts::minDeposit

Минимальный депозит для пула

Пул с алгоритмом AlgoTime не учитывает размер депозита для розыгрыша вознагражедения. Этот параметр позволит избежать копеечных депозитов 0 - нет ограничения

См. также
AlgoTime

◆ minSeconds

nat CPoolGames::t_opts::minSeconds

Минимальное время (в секундах) нахождения в пуле для участия в розыгрыше

Параметр не влияет на внесение депозита, пользователь может вносить депозит в любой момент, если проходит по другим ограничениям, он сможет участвовать в следующих розыгрышах 0 - нет ограничения. Максимальное значение - длительность партии gameSeconds
